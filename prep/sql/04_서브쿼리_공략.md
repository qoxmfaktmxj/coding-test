# SQL 04 — 서브쿼리

> "평균보다 ~", "각 그룹의 최댓값 행" 같은 요구의 정답. 스칼라 / IN / **상관 서브쿼리** 3종.

---

## 문제 1. 평균보다 비싼 상품 (스칼라 서브쿼리)

> **전체 상품 평균 가격보다 비싼** 상품의 이름과 가격을 조회하세요.

**정답 (Oracle)**
```sql
SELECT name, price
FROM PRODUCT
WHERE price > (SELECT AVG(price) FROM PRODUCT)
ORDER BY product_id;
```
**해설**
- 괄호 안 서브쿼리가 **값 하나(스칼라)**를 반환 → 그 값과 비교.
- 전체 평균 226,625보다 비싼: 노트북, 모니터.
- 평균/최대/최소 등 "전체 기준값과 비교"는 스칼라 서브쿼리.

**⚠️ 함정**: 스칼라 서브쿼리는 반드시 값 1개만 반환해야 함. 여러 행 반환하면 에러(`=` 대신 `IN` 필요).

**🔗 프로그래머스 대응**: "평균 이상/이하" 유형.

---

## 문제 2. 완료 주문을 한 회원 (IN 서브쿼리)

> **완료된 주문을 한 번이라도 한 회원**의 이름을 조회하세요.

**정답 (Oracle)**
```sql
SELECT name
FROM MEMBER
WHERE member_id IN (
    SELECT member_id FROM ORDERS WHERE status = '완료'
)
ORDER BY member_id;
```
**해설**
- 서브쿼리가 **여러 값(집합)**을 반환 → `IN`으로 포함 여부 판단.
- 결과: 김철수, 이영희, 박민수, 정하늘.
- `JOIN` + `DISTINCT`로도 가능하지만, "존재 여부"만 볼 땐 `IN`/`EXISTS`가 의도가 명확.

**⚠️ 함정**: `NOT IN`에 NULL이 섞이면 결과가 비어버림. "없는 것"은 `NOT EXISTS`나 LEFT JOIN이 안전([03](03_JOIN_공략.md)).

**🔗 프로그래머스 대응**: 서브쿼리 / EXISTS.

## 문제 3. 카테고리별 최고가 상품 (상관 서브쿼리)

> **각 카테고리에서 가장 비싼 상품**의 이름, 카테고리, 가격을
> 카테고리 순으로 조회하세요.

**정답 (Oracle)**
```sql
SELECT p.name, p.category, p.price
FROM PRODUCT p
WHERE p.price = (
    SELECT MAX(x.price)
    FROM PRODUCT x
    WHERE x.category = p.category      -- 바깥 행을 참조 (상관)
)
ORDER BY p.category;
```
**해설**
- 서브쿼리가 **바깥 쿼리의 각 행(p)을 참조** → 행마다 다시 평가되는 **상관 서브쿼리**.
- "각 그룹의 최댓값을 가진 행"의 대표 패턴. 결과: 도서=자바의정석, 식품=커피원두, 의류=청바지, 전자=노트북.

**⚠️ 함정**: 최댓값이 동점이면 그 카테고리에서 여러 행이 나옴(의도일 수도, 아닐 수도). "정확히 1개"가 필요하면 윈도우함수 `ROW_NUMBER`([05](05_윈도우함수_공략.md))가 더 정확.

**🔗 프로그래머스 대응**: "카테고리별 최고가 상품"(고득점 키트 실제 문제와 유사).

---

### ✅ 이 장 핵심
- **스칼라**(값1): `WHERE price > (SELECT AVG...)`
- **IN**(집합): 포함 여부
- **상관 서브쿼리**: 그룹별 최대/최소 행 — 바깥 행 참조가 핵심
- "그룹별 정확히 1개"는 윈도우함수가 더 깔끔 → 다음 장
