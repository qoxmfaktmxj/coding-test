# SQL 06 — 날짜 · 문자열 (⭐ Oracle)

> MySQL과 **함수 이름이 가장 많이 다른 파트**. `DATE_FORMAT`·`YEAR()`·`CONCAT`은 오라클에서 안 통함.
> Oracle 기준으로 통일해서 외우세요.

## 문제 1. 특정 월의 주문 (날짜 범위)

> **2024년 6월에 접수된 주문**의 주문번호를 주문번호 순으로 조회하세요.

**정답 (Oracle)**
```sql
-- 방법 A: 범위 비교 (인덱스 유리, 권장) — DATE 리터럴 사용
SELECT order_id
FROM ORDERS
WHERE ordered_at >= DATE '2024-06-01'
  AND ordered_at <  DATE '2024-07-01'
ORDER BY order_id;

-- 방법 B: EXTRACT
SELECT order_id
FROM ORDERS
WHERE EXTRACT(YEAR FROM ordered_at) = 2024
  AND EXTRACT(MONTH FROM ordered_at) = 6
ORDER BY order_id;
```
**해설**
- 결과: 101,102,104,105,106,107 (6월 주문. 날짜 기준이므로 취소 주문 104도 포함).
- **방법 A 권장**: 컬럼에 함수를 안 씌워 인덱스를 탈 수 있음(성능 면접 포인트).
- Oracle `DATE`엔 시간 성분이 있으므로, "6월 전체"는 `>= 6/1 AND < 7/1`이 안전(6/30 23:59도 포함).

**⚠️ 함정**
- MySQL의 `YEAR(col)=2024`는 오라클에서 `EXTRACT(YEAR FROM col)`. 그리고 컬럼에 함수 씌우면 인덱스 무력화.
- `WHERE ordered_at >= '2024-06-01'`처럼 **문자열로 직접 비교하면 NLS_DATE_FORMAT에 의존**해 에러/오작동 → `DATE '...'` 또는 `TO_DATE`로 명시.

**🔗 프로그래머스 대응**: Date 카테고리(Oracle 환경 선택).

## 문제 2. 가입 연도별 회원 수

> **가입 연도별 회원 수**를 연도 오름차순으로 조회하세요.

**정답 (Oracle)**
```sql
SELECT EXTRACT(YEAR FROM joined_at) AS join_year,
       COUNT(*) AS cnt
FROM MEMBER
GROUP BY EXTRACT(YEAR FROM joined_at)     -- 별칭 말고 식을 반복!
ORDER BY join_year;
```
**해설**
- `EXTRACT(YEAR FROM d)`로 연도 추출. 결과: 2021:1, 2022:1, 2023:2, 2024:2.
- ⭐ **Oracle은 `GROUP BY join_year`(별칭) 불가** → `GROUP BY EXTRACT(YEAR FROM joined_at)`로 식을 반복. (ORDER BY엔 별칭 OK)
- 월별이면 `TO_CHAR(joined_at, 'YYYY-MM')`으로 묶기.

**Oracle 날짜 함수 요약**
```sql
SYSDATE                              -- 현재 (MySQL NOW())
EXTRACT(YEAR|MONTH|DAY FROM d)        -- 연/월/일 (MySQL YEAR/MONTH/DAY)
TO_CHAR(d, 'YYYY-MM-DD')             -- 포맷 (MySQL DATE_FORMAT)
TO_CHAR(d, 'YYYY-MM'), TO_CHAR(d,'DY')  -- 월/요일
TO_DATE('2024-06-01','YYYY-MM-DD')   -- 문자 → 날짜
d1 - d2                              -- 일수 차 (숫자)
ADD_MONTHS(d, 3), MONTHS_BETWEEN(d1,d2)
TRUNC(d)                             -- 시간 잘라 자정으로
```

**⚠️ 함정**: 출력 형식이 지문과 다르면 오답 처리 → `TO_CHAR(col,'YYYY-MM-DD')`로 감싸기. Oracle `DATE`는 시간 성분이 숨어 있으니 비교/집계 시 `TRUNC` 고려.

**🔗 프로그래머스 대응**: Date 카테고리.

## 문제 3. 이름 조건 검색 (문자열)

> **이름이 '김'으로 시작하는** 회원의 이름과, **성을 뺀 부분(2번째 글자부터)**을 조회하세요.

**정답 (Oracle)**
```sql
SELECT name,
       SUBSTR(name, 2) AS without_surname
FROM MEMBER
WHERE name LIKE '김%'
ORDER BY member_id;
```
**해설**
- `LIKE '김%'`: '김'으로 시작(`%`=0글자 이상, `_`=정확히 1글자).
- `SUBSTR(name, 2)`: 2번째 글자부터 끝까지(1-index). 결과: 김철수 → 철수.

**Oracle 문자열 함수 요약**
```sql
a || '-' || b            -- 연결 (CONCAT은 인자 2개만 → || 권장)
SUBSTR(s, start, len)    -- 1-index
LENGTH(s)                -- 글자 수 (Oracle은 문자 기준! 바이트는 LENGTHB)
INSTR(s, 'x')            -- 'x' 위치 (없으면 0)
UPPER/LOWER, TRIM, REPLACE(s,'a','b')
LPAD(s, 5, '0'), RPAD(s, 5, ' ')
s LIKE '%book%'          -- 포함
REGEXP_LIKE(s, '^[0-9]+$')  -- 정규식 (MySQL의 s REGEXP '...')
```

**⚠️ 함정**
- 연결은 `||`. `CONCAT(a,b,c)`처럼 3개 인자 주면 에러(Oracle CONCAT은 2개).
- Oracle `LENGTH`는 **이미 글자 수**(한글도 1글자). MySQL처럼 `CHAR_LENGTH` 따로 안 씀.

**🔗 프로그래머스 대응**: String 카테고리.

---

### ✅ 이 장 핵심 (Oracle)
- 날짜 범위: **`>= DATE '시작' AND < DATE '다음달1일'`** (인덱스 + 시간성분 안전)
- 연/월 추출 `EXTRACT`, 포맷 `TO_CHAR`, **GROUP BY엔 식 반복**(별칭 불가)
- 문자 연결 `||`, 글자 수 `LENGTH`(이미 문자 기준), 정규식 `REGEXP_LIKE`
