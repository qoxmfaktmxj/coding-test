# SQL 05 — 윈도우함수 (경력직 변별력)

> "**그룹별 순위 / 그룹별 TOP-N / 누적·이동 계산**"의 정답. 신입은 몰라도 되지만 경력직은 알면 확 앞섬.
> 형식: `함수() OVER (PARTITION BY 그룹 ORDER BY 정렬)`

---

## 문제 1. 부서별 연봉 순위

> 부서가 있는 직원에 대해 **부서 안에서 연봉 순위**를 매겨,
> 이름·부서·연봉·순위를 조회하세요. (부서, 순위 순)

**정답 (Oracle)**
```sql
SELECT name, dept_id, salary,
       RANK()       OVER (PARTITION BY dept_id ORDER BY salary DESC) AS rnk,
       DENSE_RANK() OVER (PARTITION BY dept_id ORDER BY salary DESC) AS drnk
FROM EMPLOYEE
WHERE dept_id IS NOT NULL
ORDER BY dept_id, rnk;
```
**해설**
- `PARTITION BY dept_id` = 부서별로 창(window)을 나눔. `ORDER BY salary DESC` = 그 안에서 정렬.
- 개발팀: 조개발(1) → 김개발(2) → 이코딩(3).
- **RANK vs DENSE_RANK vs ROW_NUMBER**:
  | | 동점 처리 예 (100,100,90) |
  |---|---|
  | `ROW_NUMBER()` | 1,2,3 (동점도 강제 순번) |
  | `RANK()` | 1,1,3 (다음 순위 건너뜀) |
  | `DENSE_RANK()` | 1,1,2 (안 건너뜀) |

**⚠️ 함정**: 순위 요구인데 `LIMIT`이나 서브쿼리로 우회하면 그룹별 처리가 지저분해짐. 윈도우함수가 정석.

## 문제 2. 부서별 연봉 1위 (그룹별 TOP-N)

> **각 부서에서 연봉이 가장 높은 직원 1명씩**을 조회하세요. (부서 순)

**정답 (Oracle)**
```sql
SELECT name, dept_id, salary
FROM (
    SELECT name, dept_id, salary,
           ROW_NUMBER() OVER (PARTITION BY dept_id ORDER BY salary DESC) AS rn
    FROM EMPLOYEE
    WHERE dept_id IS NOT NULL
) t
WHERE rn = 1
ORDER BY dept_id;
```
**해설**
- 윈도우함수는 `WHERE`에서 바로 못 씀(실행순서상 SELECT 이후). → **서브쿼리로 감싼 뒤 바깥에서 `rn = 1`**.
- 결과: 개발=조개발, 영업=윤영업, 인사=최인사.
- **이 "감싸서 rn=1" 패턴이 그룹별 TOP-N의 표준 정답.** TOP-3면 `rn <= 3`.

**⚠️ 함정**: 동점 1위가 둘인데 "정확히 1명"이 필요하면 `ORDER BY salary DESC, emp_id`처럼 tie-breaker를 넣어야 결정적.

## 문제 3. 부서 평균 대비 내 연봉 (집계 윈도우)

> 각 직원의 연봉이 **자기 부서 평균 연봉**과 얼마나 차이 나는지 조회하세요.

**정답 (Oracle)**
```sql
SELECT name, dept_id, salary,
       AVG(salary) OVER (PARTITION BY dept_id)        AS dept_avg,
       salary - AVG(salary) OVER (PARTITION BY dept_id) AS diff
FROM EMPLOYEE
WHERE dept_id IS NOT NULL
ORDER BY dept_id, salary DESC;
```
**해설**
- 집계함수를 `OVER`와 함께 쓰면 **행을 줄이지 않고** 그룹 집계값을 각 행에 붙임(GROUP BY와 다름).
- 개발팀 평균 6,166,666 → 조개발은 +1,833,334, 이코딩은 -1,166,666.
- 이렇게 "개별 행 + 그룹 통계"를 한 번에 보여줘야 할 때 GROUP BY로는 불가, 윈도우가 유일.

**⚠️ 함정**: `GROUP BY`는 행을 그룹당 1개로 접지만, 윈도우 집계는 원본 행 수를 유지. 목적에 맞게 선택.

---

### ✅ 이 장 핵심
- `함수() OVER (PARTITION BY 그룹 ORDER BY 정렬)`
- 순위 3종: ROW_NUMBER / RANK / DENSE_RANK (동점 처리 차이)
- **그룹별 TOP-N = 서브쿼리로 감싸고 `rn <= N`**
- 집계 윈도우는 행을 안 접고 그룹 통계를 붙임
