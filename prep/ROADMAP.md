- 플랫폼: **프로그래머스 온라인**, `class Solution` 제출형 (지금 `code_pack`이랑 동일)
- 구성: **SQL 1문제 + 알고리즘 2문제 (총 3문제)**, 난이도 **프로그래머스 Lv.2~3**
- **SQL DBMS = Oracle** ⭐ (MySQL 문법 `LIMIT`/`IFNULL`/`DATE_FORMAT` 쓰면 에러 — [SQL 치트시트](sql/00_SQL_치트시트.md) 참고)
- 제한시간: **약 120분 (추정)** — 셋 다 풀기엔 빡셈. **SQL에서 시간 빨리는 게 최대 함정**
- 채점: **예시 테스트케이스만 보이고, 제출 후 점수 비공개** → *내가 반례를 직접 만들어 검증해야 함*
- 경력직: **SQL 비중 체감이 큼**. 알고리즘은 "정확히 1개 완벽 + 1개 부분"만 해도 경쟁력.

```
120분 기준
├─ 00~05분 : 3문제 전부 읽고 난이도 스캔 → 쉬운 순서 정하기
├─ 05~35분 : SQL 먼저 (가장 확실한 득점, 30분 컷. 안 풀리면 잠깐 미루기)

```
> ⚠️ 롯데는 점수를 안 보여줌. "제출했더니 통과처럼 보임"에 속지 말 것. **엣지케이스는 내 머리로 설계.**

## 3. 알고리즘 유형 "인식 → 대응" 초압축표

> 문제를 읽고 **어떤 신호가 보이면 어떤 유형인지** 즉시 판단하는 게 핵심.
> 상세 템플릿·해설은 → [algo/00_알고리즘_치트시트.md](algo/00_알고리즘_치트시트.md)

| 문제에 이런 신호가… | → 이 유형 | 한 줄 대응 |
|---|---|---|
| "최소 이동 횟수", 격자, 모든 간선 비용=1 | **BFS** | 큐 + 방문거리 배열 |
| "연결된 덩어리 수", 영역, 미로 채우기 | **DFS/flood fill** | 재귀 or 스택, visited |
| "비용/가중치가 다른 최단거리" | **다익스트라** | PriorityQueue + dist[] |
| "최댓값/최솟값 만들기", 정렬 후 하나씩 선택 | **그리디** | 정렬 기준 설계가 핵심 |
| "경우의 수 다 세기", N≤20, 순열/조합 | **백트래킹** | 선택→재귀→되돌리기 |
| "이전 결과로 현재 결과", 겹치는 부분문제 | **DP** | 점화식 + 메모이제이션 |
| "무게/용량 한정, 가치 최대" | **배낭 DP** | dp[용량] 갱신 |
| "구간 합 반복 조회" | **누적합** | prefix[i] 미리 계산 |
| "정렬된 것에서 값/조건 찾기", "최대를 최소로" | **이분탐색** | lower_bound / 파라메트릭 서치 |
| N≤20인데 "부분집합/방문상태 저장" | **비트마스크** | 상태를 int 비트로 |
| "격자를 규칙대로 채우기", 나선/삼각형 | **도형·구현** | 방향배열 + 시뮬레이션 |
| 시키는 대로 그대로 처리, 규칙 복잡 | **구현·시뮬** | 자료구조 정직하게 옮기기 |

## 4. SQL 핵심 (경력직 최우선)

⭐ **DBMS는 Oracle.** 상세는 → [sql/00_SQL_치트시트.md](sql/00_SQL_치트시트.md), [sql/README.md](sql/README.md)

**꼭 몸에 붙여야 할 7개 패턴 (Oracle):**
1. `JOIN` — 없는 행까지 보려면 `LEFT JOIN` + `IS NULL`
2. `GROUP BY … HAVING` — 집계 후 조건은 `HAVING`. **GROUP BY엔 별칭 불가(식 반복)**
3. 상관 서브쿼리 — "각 그룹의 최댓값 행"
4. **윈도우함수** — `ROW_NUMBER()/RANK() OVER(PARTITION BY … ORDER BY …)` → 그룹별 TOP-N
5. **상위 N개** — `LIMIT` 없음! 정렬한 인라인뷰 + `ROWNUM <= N`, 또는 `FETCH FIRST N ROWS ONLY`(12c+)
6. 날짜/문자 — `TO_CHAR`, `EXTRACT`, `TO_DATE`, `||`(연결), `NVL`, `DECODE`, `REGEXP_LIKE`
7. `NULL` → `IS NULL` / `NVL(COALESCE)` / `CASE WHEN`

> ⚠️ MySQL 습관(`LIMIT`,`IFNULL`,`DATE_FORMAT`,`YEAR()`,`CONCAT` 다인자) 그대로 쓰면 **오라클에서 에러**.
> ⏱ SQL은 "정답을 한 번에" 나와야 함 → **문법 암기 < 손 연습**.
