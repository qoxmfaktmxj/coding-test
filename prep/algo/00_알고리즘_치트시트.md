## 2. 구현·시뮬레이션

**🔎 언제**: "시키는 규칙대로 그대로 처리", 로봇 이동, 시계방향 회전, 시간 흐름 처리. 특별한 알고리즘 없이 **정확한 상태 관리**가 전부.

**🧭 접근**: (1) 상태를 무엇으로 표현할지 먼저 정한다 (2) 규칙을 함수로 잘게 쪼갠다 (3) 한 스텝씩 손으로 따라가며 검증.

**⚠️ 함정**: 경계 처리, 1-index/0-index 혼동, 회전/방향 순서. 규칙을 "대충 기억"하지 말고 문제 그대로 옮길 것.

```java
// 예: 방향 유지하며 이동, 벽 만나면 우회전
int dir = 0; // 0=상,1=우,2=하,3=좌 등 문제 정의대로
int[] dr = {-1,0,1,0}, dc = {0,1,0,-1};
int nr = r + dr[dir], nc = c + dc[dir];
if (out(nr,nc) || wall) dir = (dir+1) % 4; // 우회전
else { r = nr; c = nc; }
```

## 3. BFS

**🔎 언제**: "**최소 이동 횟수/최단 거리**"인데 **모든 이동 비용이 동일(=1)**. 격자·미로·전파(불/바이러스)·레벨 단위 확산.

**🧭 접근**: 시작점을 큐에 넣고 거리 0 → 큐에서 꺼내 4방향 확장, 처음 도달할 때가 최단.

**⚠️ 함정**: 방문 표시는 **큐에 넣을 때** 해야 중복 방지. 거리 배열을 방문표시로 겸하면 깔끔.

```java
int bfs(int[][] maps) {
    int R = maps.length, C = maps[0].length;
    int[][] dist = new int[R][C];
    for (int[] row : dist) Arrays.fill(row, -1);
    ArrayDeque<int[]> q = new ArrayDeque<>();
    q.offer(new int[]{0,0}); dist[0][0] = 0;
    int[] dr={-1,1,0,0}, dc={0,0,-1,1};
    while (!q.isEmpty()) {
        int[] c = q.poll();
        for (int d=0; d<4; d++) {
            int nr=c[0]+dr[d], nc=c[1]+dc[d];
            if (nr<0||nr>=R||nc<0||nc>=C) continue;
            if (maps[nr][nc]==0 || dist[nr][nc]!=-1) continue;
            dist[nr][nc] = dist[c[0]][c[1]] + 1;
            q.offer(new int[]{nr,nc});
        }
    }
    return dist[R-1][C-1];
}
```
**시간복잡도**: O(V+E). 격자면 O(R·C).

## 4. DFS / flood fill

**🔎 언제**: "**연결된 덩어리 개수**", 영역 크기, 섬 개수, 경로 존재 여부. 깊이 우선으로 끝까지 파고듦.

**🧭 접근**: 방문 안 한 칸에서 DFS 시작 → 재귀로 사방 전파 → 한 번의 DFS = 한 덩어리.

**⚠️ 함정**: 격자가 크면 재귀 깊이 스택오버플로 → 스택 자료구조로 전환 고려. 방문 체크 필수.

```java
int R, C; int[][] g; boolean[][] vis;
void dfs(int r, int c) {
    vis[r][c] = true;
    int[] dr={-1,1,0,0}, dc={0,0,-1,1};
    for (int d=0; d<4; d++) {
        int nr=r+dr[d], nc=c+dc[d];
        if (nr<0||nr>=R||nc<0||nc>=C) continue;
        if (g[nr][nc]==0 || vis[nr][nc]) continue;
        dfs(nr, nc);
    }
}
// 덩어리 수 세기
int count = 0;
for (int i=0;i<R;i++) for (int j=0;j<C;j++)
    if (g[i][j]==1 && !vis[i][j]) { dfs(i,j); count++; }
```

## 5. 정렬·그리디

**🔎 언제**: "최대/최소로 만들기", "가장 많이/적게", 회의실 배정, 동전, 마감 정렬. **매 순간 최선을 고르면 전체 최선**이 성립할 때.

**🧭 접근**: **정렬 기준 설계가 90%**. "무엇을 기준으로 정렬하면 탐욕이 성립하나?"를 먼저 증명하듯 생각.

**⚠️ 함정**: 그리디가 성립하는지 반례로 의심. 안 되면 DP. 정수 오버플로(합계는 `long`).

```java
// 예: 회의 끝나는 시간 오름차순 → 겹치지 않게 최대 선택
Arrays.sort(meet, (a,b) -> a[1]-b[1]); // 끝나는 시간 기준
int end = -1, cnt = 0;
for (int[] m : meet)
    if (m[0] >= end) { end = m[1]; cnt++; }

// Comparator 국룰: 오름차순 a-b, 내림차순 b-a (오버플로 주의 시 Integer.compare)
```

## 6. 우선순위 큐

**🔎 언제**: "매번 가장 급한/작은/큰 것을 꺼낸다". K번째, 실시간 최소/최대, 작업 스케줄링, 병합.

**🧭 접근**: 최소힙 기본. 최대힙은 `Collections.reverseOrder()` 또는 `(a,b)->b-a`.

**⚠️ 함정**: `poll()` 후 유효성 재검사(지연 삭제 패턴). 커스텀 정렬은 `Comparator` 명확히.

```java
PriorityQueue<int[]> pq = new PriorityQueue<>(
    (a,b) -> a[0]!=b[0] ? a[0]-b[0] : a[1]-b[1]); // 1순위 a[0], 2순위 a[1]
pq.offer(new int[]{cost, node});
int[] top = pq.poll();
```

## 7. 다익스트라

**🔎 언제**: "최단 거리/최소 비용"인데 **간선마다 비용이 다름**(양수). BFS로 안 되는 가중 그래프.

**🧭 접근**: `dist[]`를 INF로 초기화 → 시작점 0 → PQ에서 최소 비용 노드 꺼내 완화(relax).

**⚠️ 함정**: 음수 간선이면 다익스트라 불가(벨만포드). `dist`는 `long` 고려. 꺼낸 게 이미 갱신된 값보다 크면 skip.

```java
int[] dijkstra(List<int[]>[] adj, int start, int n) {
    int[] dist = new int[n];
    Arrays.fill(dist, Integer.MAX_VALUE);
    dist[start] = 0;
    PriorityQueue<int[]> pq = new PriorityQueue<>((a,b)->a[1]-b[1]);
    pq.offer(new int[]{start, 0});
    while (!pq.isEmpty()) {
        int[] cur = pq.poll();
        int u = cur[0], d = cur[1];
        if (d > dist[u]) continue;          // 오래된 값 skip
        for (int[] e : adj[u]) {            // e = {v, w}
            int v = e[0], w = e[1];
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w;
                pq.offer(new int[]{v, dist[v]});
            }
        }
    }
    return dist;
}
```
**시간복잡도**: O(E log V).

## 8. DP

**🔎 언제**: "이전 답으로 현재 답", 겹치는 부분문제 + 최적 부분구조. 계단·피보나치·최장부분수열·경로 수·최소 비용.

**🧭 접근**: (1) 상태 정의 `dp[i]=…` (2) 점화식 (3) 초기값 (4) 계산 순서. **"dp[i]가 뭘 의미하는지" 한 문장으로 말할 수 있어야 함.**

**⚠️ 함정**: 상태 정의 모호하면 다 틀림. 배열 크기/초기값. 합은 `long`.

```java
// 예: 1D DP — 계단 오르기 (1칸/2칸)
long[] dp = new long[n+1];
dp[0] = 1; dp[1] = 1;
for (int i=2; i<=n; i++) dp[i] = dp[i-1] + dp[i-2];

// 예: 2D DP — 격자 경로 최소합
int[][] dp = new int[R][C];
dp[0][0] = g[0][0];
for (int i=0;i<R;i++) for (int j=0;j<C;j++) {
    if (i==0 && j==0) continue;
    int best = Integer.MAX_VALUE;
    if (i>0) best = Math.min(best, dp[i-1][j]);
    if (j>0) best = Math.min(best, dp[i][j-1]);
    dp[i][j] = best + g[i][j];
}
```

## 9. 배낭 DP

**🔎 언제**: "무게/용량 한정, 가치 최대". 0/1 배낭(각 물건 1개), 부분합 만들기.

**🧭 접근**: `dp[w]` = 용량 w일 때 최대 가치. **물건 루프 안에서 용량은 역순**(0/1 배낭 핵심).

**⚠️ 함정**: 용량 정순으로 돌면 같은 물건 중복 사용됨(그건 무한 배낭). 0/1은 반드시 역순.

```java
int[] dp = new int[cap+1];
for (int[] item : items) {        // item = {weight, value}
    int wt = item[0], val = item[1];
    for (int w = cap; w >= wt; w--)       // 역순!
        dp[w] = Math.max(dp[w], dp[w-wt] + val);
}
return dp[cap];
```

## 10. 백트래킹

**🔎 언제**: "모든 경우의 수", 순열·조합·부분집합, N-Queen, 조건 만족 조합 찾기. **N이 작음(≤ 약 20)**.

**🧭 접근**: 선택 → 재귀 → **되돌리기(undo)**. 가지치기(pruning)로 불가능한 가지 조기 차단.

**⚠️ 함정**: 되돌리기 누락 → 오답. 방문/사용 배열 상태 복원 필수. 지수 시간이라 N 확인.

```java
// 조합: n개 중 r개 고르기
void comb(int start, int n, int r, List<Integer> pick, List<List<Integer>> res) {
    if (pick.size() == r) { res.add(new ArrayList<>(pick)); return; }
    for (int i = start; i < n; i++) {
        pick.add(i);
        comb(i + 1, n, r, pick, res);
        pick.remove(pick.size() - 1);   // 되돌리기
    }
}

// 순열: used[] 사용
void perm(int[] arr, boolean[] used, List<Integer> cur, List<List<Integer>> res) {
    if (cur.size() == arr.length) { res.add(new ArrayList<>(cur)); return; }
    for (int i = 0; i < arr.length; i++) {
        if (used[i]) continue;
        used[i] = true; cur.add(arr[i]);
        perm(arr, used, cur, res);
        used[i] = false; cur.remove(cur.size()-1);   // 되돌리기
    }
}
```

## 11. 누적합

**🔎 언제**: "구간 [l,r] 합을 여러 번 조회", 2D 부분합, 이동평균. 미리 계산해두고 O(1) 조회.

**🧭 접근**: `prefix[i] = prefix[i-1] + a[i]`. 구간합 = `prefix[r] - prefix[l-1]`.

**⚠️ 함정**: 인덱스 off-by-one. 합은 `long`.

```java
long[] pre = new long[n+1];
for (int i=1; i<=n; i++) pre[i] = pre[i-1] + a[i-1];
long sum = pre[r] - pre[l-1];   // 1-indexed 구간 [l,r]

// 2D 누적합
long[][] p = new long[R+1][C+1];
for (int i=1;i<=R;i++) for (int j=1;j<=C;j++)
    p[i][j] = g[i-1][j-1] + p[i-1][j] + p[i][j-1] - p[i-1][j-1];
```

## 12. 이분탐색 / 파라메트릭 서치

**🔎 언제**: 정렬된 배열에서 값 찾기. 또는 **"~을 최대/최소로 하는 최적값"**(파라메트릭): 랜선 자르기, 예산, 최소 최대거리.

**🧭 접근**: 정답을 이분. `check(mid)`가 만족하면 범위를 좁힘. "가능/불가능"이 단조로울 때만.

**⚠️ 함정**: `mid = lo + (hi-lo)/2` (오버플로 방지). lo/hi 갱신 방향, 종료 조건.

```java
// lower_bound: target 이상이 처음 나오는 위치
int lowerBound(int[] a, int target) {
    int lo = 0, hi = a.length;
    while (lo < hi) {
        int mid = lo + (hi - lo) / 2;
        if (a[mid] < target) lo = mid + 1;
        else hi = mid;
    }
    return lo;
}

// 파라메트릭: "조건 만족하는 최댓값" 찾기
int lo = 1, hi = MAX, ans = 0;
while (lo <= hi) {
    int mid = lo + (hi - lo) / 2;
    if (possible(mid)) { ans = mid; lo = mid + 1; }
    else hi = mid - 1;
}
```

## 13. 비트마스크

**🔎 언제**: "부분집합/방문상태를 저장", 외판원(TSP), 상태압축 DP. **원소 수 N ≤ 약 20**(2^20 ≈ 100만).

**🧭 접근**: 집합을 정수 비트로. i번째 포함 여부 = `(mask >> i) & 1`.

**⚠️ 함정**: 연산자 우선순위(`&`는 `==`보다 낮음 → 괄호). `1 << i`에서 i 크면 `1L`.

```java
int full = (1 << n) - 1;              // 전체 집합
boolean has = (mask & (1 << i)) != 0; // i 포함?
int add = mask | (1 << i);            // i 추가
int del = mask & ~(1 << i);           // i 제거
int bits = Integer.bitCount(mask);    // 켜진 비트 수

// 상태압축 DP (방문 위치 + 현재 노드)
// dp[mask][cur] = 지금까지 방문 mask, 현재 cur일 때 최소비용
```

## 14. Union-Find (보너스)

**🔎 언제**: "같은 그룹인가", 사이클 판별, 연결 요소, 크루스칼 MST. 도형/그래프 문제에서 종종 유용.

```java
int[] parent;
int find(int x) { return parent[x]==x ? x : (parent[x]=find(parent[x])); }
void union(int a, int b) { parent[find(a)] = find(b); }
// 초기화: for(i) parent[i]=i;
```

---

> ✅ **시험 직전 3분 루틴**: (1) 방향배열 손코딩 (2) BFS/DFS 뼈대 (3) 다익스트라 PQ 정렬 방향 (4) 0/1 배낭 "역순" (5) 백트래킹 "되돌리기".
> 이 5개만 손이 기억하면 Lv.2~3 알고리즘 2문제 중 1~1.5개는 확보.
