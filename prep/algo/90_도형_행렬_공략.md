# 🔷 도형·행렬 구현 공략 (LE 단골)

> 후기에서 "**Matrix 숫자 채우기, 삼각형, 피라미드**" 유형이 반복 언급됨.
> 특별한 알고리즘이 아니라 **방향 제어 + 경계 처리**가 전부인 구현 문제.
> 모든 코드는 프로그래머스 `class Solution` 제출형. 아래 3개 답지는 **실제 컴파일·실행으로 출력 검증 완료**.

## 공통 무기: 방향 배열 + "벽이면 꺾기"
```java
int[] dr = {0, 1, 0, -1};  // 우, 하, 좌, 상 (시계방향)
int[] dc = {1, 0, -1, 0};
// 다음 칸이 격자 밖이거나 이미 채워졌으면 dir = (dir+1) % 4
```
이 한 패턴이 나선형·달팽이·회전 문제의 90%를 커버.

## 문제 1. 나선형 행렬 (Spiral)

> 정수 `n`이 주어질 때, `n × n` 행렬을 **1부터 n²까지 시계방향 나선형**으로 채워 반환하세요.
> 예: n=4
> ```
>  1  2  3  4
> 12 13 14  5
> 11 16 15  6
> 10  9  8  7
> ```

**정답 (검증됨)**
```java
class Solution {
    public int[][] solution(int n) {
        int[][] a = new int[n][n];
        int[] dr = {0, 1, 0, -1};   // 우,하,좌,상
        int[] dc = {1, 0, -1, 0};
        int r = 0, c = 0, dir = 0;
        for (int v = 1; v <= n * n; v++) {
            a[r][c] = v;                        // 현재 칸 채우기
            int nr = r + dr[dir], nc = c + dc[dir];
            if (nr < 0 || nr >= n || nc < 0 || nc >= n || a[nr][nc] != 0) {
                dir = (dir + 1) % 4;            // 벽 or 이미 채움 => 시계방향 꺾기
                nr = r + dr[dir]; nc = c + dc[dir];
            }
            r = nr; c = nc;                     // 이동
        }
        return a;
    }
}
```
**해설 (읽기용)**
- 값 `v`를 1→n²까지 하나씩 놓으면서 **현재 방향으로 전진**.
- 전진할 칸이 **격자 밖**이거나 **이미 채워진 칸(≠0)**이면 방향을 시계방향으로 한 번 꺾음.
- `a[nr][nc] != 0` 체크가 핵심 — 이게 나선의 "안쪽으로 감기"를 만든다.
- 방향 순서를 `{우,하,좌,상}`으로 두면 시계방향. 반시계면 순서만 바꾸면 됨.

**⚠️ 함정**
- 마지막 값(n²)을 놓은 뒤에도 이동을 시도하는데, 루프가 끝나므로 문제없음. 단 `v <= n*n` 경계 확인.
- 0을 "빈 칸" 표식으로 쓰므로 채우는 값은 1부터. 0부터 채우면 이 방식이 깨짐(빈칸 구분 불가) → `visited` 배열 별도 사용.

## 문제 2. 파스칼 삼각형

> 정수 `n`이 주어질 때, **파스칼 삼각형 n행**을 반환하세요(i행은 i+1개 원소).
> 각 원소는 바로 위 두 수의 합, 양 끝은 1.
> 예: n=5 → `[[1],[1,1],[1,2,1],[1,3,3,1],[1,4,6,4,1]]`

**정답 (검증됨)**
```java
class Solution {
    public int[][] solution(int n) {
        int[][] t = new int[n][];
        for (int i = 0; i < n; i++) {
            t[i] = new int[i + 1];              // i행은 i+1칸 (가변 길이)
            t[i][0] = t[i][i] = 1;              // 양 끝은 1
            for (int j = 1; j < i; j++)
                t[i][j] = t[i - 1][j - 1] + t[i - 1][j];  // 위 두 수 합
        }
        return t;
    }
}
```
**해설 (읽기용)**
- **가변 길이 2차원 배열**(`new int[n][]` 후 행마다 크기 지정)이 포인트 — 직사각형 배열이 아님.
- 점화식 `t[i][j] = t[i-1][j-1] + t[i-1][j]`는 사실 **DP**. 도형이지만 DP 사고가 들어감.
- 양 끝(`j=0`, `j=i`)은 1로 먼저 세팅하고, 내부만 합으로 채움.

**⚠️ 함정**
- `t[i-1][...]` 접근 시 i=0행에서 내부 루프(`j<i`)가 안 도므로 인덱스 에러 없음. 경계 자연 처리.
- 값이 커지면(파스칼은 조합수라 폭발) `long` 필요. n이 크면 오버플로 주의.

## 문제 3. 지그재그(뱀) 행렬

> 정수 `n, m`이 주어질 때, `n × m` 행렬을 **1부터 뱀처럼 지그재그**로 채워 반환하세요.
> 짝수 행(0-index)은 왼→오, 홀수 행은 오→왼.
> 예: n=3, m=4
> ```
> 1  2  3  4
> 8  7  6  5
> 9 10 11 12
> ```

**정답 (검증됨)**
```java
class Solution {
    public int[][] solution(int n, int m) {
        int[][] a = new int[n][m];
        int v = 1;
        for (int i = 0; i < n; i++) {
            if (i % 2 == 0)                     // 짝수 행: 왼 -> 오
                for (int j = 0; j < m; j++) a[i][j] = v++;
            else                                // 홀수 행: 오 -> 왼
                for (int j = m - 1; j >= 0; j--) a[i][j] = v++;
        }
        return a;
    }
}
```
**해설 (읽기용)**
- 행 번호의 **홀짝으로 방향만 뒤집으면** 지그재그 완성. 방향배열까지 갈 필요 없음.
- `v++`는 "현재 v를 넣고 v를 1 증가" — 채우기 카운터의 국룰.

**⚠️ 함정**
- "열 방향" 지그재그(세로로 뱀)면 i/j 역할을 바꾸면 됨. 지문에서 방향을 꼭 확인.
- 대각선 지그재그(예: 지그재그 변환) 유형은 규칙이 다르니 별개.
