## Q3. (알고리즘) 미로 최단 거리

> `0`(벽)과 `1`(길)로 이루어진 `maps`(`int[][]`)에서, 좌상단 `(0,0)`에서 우하단 `(R-1,C-1)`까지
> 상하좌우로 이동하는 **최소 이동 칸 수**를 반환하세요. 도달 불가면 `-1`.

제한: 1 ≤ R,C ≤ 100. 각 칸 이동 비용 동일.

## ── 정답 ──

### A1. Q1 SQL (Oracle · 결과 검증됨)
```sql
SELECT category, total_sales, rnk
FROM (
    SELECT p.category,
           SUM(oi.quantity * p.price) AS total_sales,
           RANK() OVER (ORDER BY SUM(oi.quantity * p.price) DESC) AS rnk
    FROM ORDERS o
    JOIN ORDER_ITEM oi ON o.order_id = oi.order_id
    JOIN PRODUCT p     ON oi.product_id = p.product_id
    WHERE o.status = '완료'
    GROUP BY p.category
) t
WHERE rnk <= 2
ORDER BY rnk;
```
- 결과: 전자(3,017,000, 1위), 도서(102,000, 2위).
- 포인트: **집계(SUM) 결과에 윈도우함수(RANK)를 얹는** 2단 구조. 서브쿼리로 감싸 `rnk<=2` 필터.
- 자세한 원리 → [SQL 05 윈도우](sql/05_윈도우함수_공략.md), [SQL 03 JOIN](sql/03_JOIN_공략.md).
